# Outlet Intelligence Dashboard

Next.js 16 web app for the DataStorm 7.0 beverage distribution challenge. It displays outlet potential, budget allocation data, and Western Province map intelligence from static JSON files generated by the Python pipeline.

## Quick Start

From the repository root:

```bash
pnpm install --prefix web
pnpm run dev --prefix web
```

Open [http://localhost:3000](http://localhost:3000).

From inside `web/`:

```bash
pnpm install
pnpm run dev
```

## Scripts

```bash
pnpm run dev        # local development server
pnpm run build      # production build
pnpm run start      # serve production build
pnpm run lint       # eslint
pnpm run format     # prettier
pnpm run typecheck  # tsc --noEmit
```

## Current Routes

| Route | Purpose |
|---|---|
| `/` | Overview dashboard and outlet data table |
| `/map` | Western Province outlet map with clustered Leaflet markers |
| `/settings` | Data connection, AI, and pipeline configuration surface |
| `/api/chat` | AI chat endpoint |
| `/api/outlet-insight` | Outlet insight endpoint |

## Data Files

The app reads static files from `web/public/data/`. These are generated by phase 9 of the root pipeline:

```bash
./run_pipeline.sh --start-from 9
```

| File | Rows | Source |
|---|---:|---|
| `public/data/outlet_coordinates.json` | 20,000 | Pipeline export from cleaned coordinates |
| `public/data/predictions.json` | 20,000 | `data/predictions/ctrl_freaks_predictions.csv` |
| `public/data/budget_allocations.json` | 1,799 | `data/budget/ctrl_freaks_budget_mapping.csv` |
| `public/data/outlets.json` | 20,000 | merged predictions, coordinates, master data, distributor, and spend |
| `public/data/analysis_manifest.json` | 4 datasets | generated asset manifest |

## Stack

- **Framework:** Next.js 16 App Router + React 19 + TypeScript
- **UI:** shadcn-style components, Base UI, lucide-react, Tabler icons
- **Map:** Leaflet + react-leaflet + react-leaflet-cluster
- **Tables/charts:** @tanstack/react-table + Recharts
- **Package manager:** pnpm

## Architecture Notes

- Static data is baked into the app from `public/data/`; no data warehouse is needed to run the demo.
- The map is client-rendered to avoid SSR issues with Leaflet.
- `components/outlet-map.tsx` renders clustered outlet markers and popups.
- `components/data-table.tsx` powers the sortable/paginated outlet table.
- `app/api/chat/route.ts` and `app/api/outlet-insight/route.ts` provide AI-facing endpoints for outlet explanations and chat-style interaction.
