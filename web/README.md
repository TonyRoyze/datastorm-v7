# Outlet Intelligence Dashboard

Next.js 16 web app for the DataStorm 7.0 beverage distribution challenge. Displays latent demand predictions and budget allocations on an interactive map + dashboard.

## Pages

| Route | Page | Content |
|-------|------|---------|
| `/` | Overview | 6 metric cards + strategy summary + budget DataTable |
| `/map` | Outlet Map | Western Province ~8,900 outlets, clustered CircleMarkers with volume/size/color popups |
| `/budget` | Budget Allocation | Sorted table of 1,799 allocations with LKR + share % |
| `/settings` | Settings | Kafka streaming config, GenAI API keys, pipeline settings |

## Quick Start

```bash
pnpm install
```

Data files in `public/data/` are generated by the main pipeline (phase 9). To use mock data, copy sample files there.

```bash
pnpm run dev        # → http://localhost:3000
pnpm run build      # production build
pnpm run start      # serve production build
pnpm run lint       # eslint
pnpm run format     # prettier
pnpm run typecheck  # tsc --noEmit
```

## Stack

- **Framework**: Next.js 16 (App Router) + TypeScript
- **UI**: shadcn/ui + inline styles + lucide-react
- **Map**: react-leaflet (`@shadcn-map/map`) + MarkerClusterGroup with `chunkedLoading`
- **Table**: @tanstack/react-table with pagination, sorting, row selection
- **Icons**: lucide-react
- **Package manager**: pnpm

## Architecture

- 4 static routes (no API routes — data baked in at build time from `public/data/` JSON files)
- Map mounted once in `AppShell` (persistent layout), hidden off-screen on non-map pages via `position: absolute; left: -9999px`. Instant navigation — no Leaflet re-initialization.
- Sidebar dashboard layout with shadcn `SidebarProvider`, `AppSidebar`, `SidebarInset`, `SiteHeader`
- Bring your own data via **Kafka streaming** or **data warehouse connection** — configure brokers, topics, and credentials in the Settings page (`/settings`)
